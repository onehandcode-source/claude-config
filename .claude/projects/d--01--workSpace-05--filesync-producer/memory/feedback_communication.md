---
name: 커뮤니케이션 방식
description: 사용자와의 상호작용 스타일 및 선호도
type: feedback
---

한국어로만 답변한다. 사용자가 세션 초반에 "한글로 대답해"를 매번 요청함.

**Why:** 사용자가 한국어 사용자이며, 영어 답변이 기본값으로 나오는 경우 매번 수동 수정 요청이 필요해 비효율적.

**How to apply:** 모든 설명, 요약, 오류 분석, 진행 상황 보고를 한국어로 작성. 코드 식별자(클래스명, 변수명, 어노테이션 등)는 영어 원문 유지.

---

사용자는 설명보다 **실행 결과 기반** 확인을 선호한다. 이론적 분석 후 "직접 run 해서 로그 확인해", "다시 실행시켜서 확인해"처럼 실제 동작 검증을 요청함.

**Why:** 코드 분석만으로는 확신하기 어렵고, 실행 로그로 직접 확인하는 것이 더 신뢰할 수 있다는 판단.

**How to apply:** 버그 수정 제안 시 이론적 설명 후 즉시 빌드 및 실행 검증까지 수행. 결과를 로그에서 구체적인 수치/쿼리로 확인해서 보고.
